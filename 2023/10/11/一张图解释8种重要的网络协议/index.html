​ 网络协议是在网络中的2台计算机之间相互传递数据的标准方式。

1. HTTP (超文本传输协议)

​ HTTP 是一个能够获取资源（例如HTML文件）的协议，是Web上任何数据交换的基础，而且是Client-server协议。

2. HTTP/3

​ HTTP/3是HTTP的下一个主要版本。它运行在QUIC上，QUIC是一种新的传输协议，专为大量移动互联网使用设计。它依赖UDP协议，而不是TCP协议，使得网页响应速度加快。VR软件需要更多的带宽来渲染复杂的虚拟场景里的细节，并且很可能从迁移到QUIC支持的HTTP/3中受益。

3. HTTPS (超文本传输安全协议)

​ HTTPS扩展了HTTP，并使用加密来进行安全通信。

4. WebSocket

​ WebSocket是一个在TCP上提供全双工通信的协议。客户端建立WebSocket来接收来自后端服务的实时更新。与总是“Pull”数据的REST不同，WebSocket允许“Push”数据。在线游戏、股票交易和消息传递等应用程序利用WebSocket进行实时通信。

5. TCP (传输控制协议)

​ TCP被设计用来在Internet上发送数据包，确保在网络上成功传递数据和消息。许多应用层协议建立在TCP之上。

6. UDP (用户数据报协议)

​ UDP协议将数据包直接发送到目标计算机，而不需要先建立连接。UDP通常用于对时间敏感的通信，在这种通信中，偶尔丢包比等待要好。语音和视频流量通常使用该协议发送。

7. SMTP （简单邮件传输协议）

​ SMTP是一种用于从一个用户输送Email到另外一个用户的标准协议

8. FTP （文件传输协议）

​ FTP协议用于在客户端和服务器之间传输计算机文件。它具有控制通道和数据通道的独立连接。
